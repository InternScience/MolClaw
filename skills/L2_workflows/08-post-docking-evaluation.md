# Post-Docking Evaluation and Consensus Ranking Workflow

## Applicability

**Use this skill when:** You already have docking result files (poses, scores) from ANY source and need systematic multi-method evaluation, rescoring, interaction analysis, or consensus ranking.

**Do NOT use this skill when:** You have not yet performed docking (use Skill 2). This skill is for **deeper analysis** of existing results.

## Prerequisites

| Input | Source | Required? |
|-------|--------|-----------|
| Docking pose files | Skill 2 output; or external PDBQT/SDF/PDB | Yes |
| Receptor PDB | Skill 1 `prepared_pdb` | Yes |
| Docking scores | From the docking tool | Recommended |
| Numbering scheme info | Skill 1 `numbering_scheme` | Recommended (for residue mapping) |

## Module 1: EquiScore Rescoring

EquiScore is an SE(3)-equivariant graph neural network scoring function, independent of Vina.

**Workflow:** `equiscore_pocket` (extract pocket + split ligand) → `equiscore_screen` (score).

**EquiScore interpretation:** Score range 0–1; higher = better. > 0.7: high-confidence active; 0.4–0.7: moderate; < 0.4: low confidence.

**Multi-pose mode:** For ≤ 50 candidates, use `multi_pose=True, pose_num=3`. For larger sets, use `multi_pose=False`.

**Format note:** If docking results are in PDBQT format, EquiScore's pocket extraction step will generate SDF files. Save the `single_sdf_dir` path — interaction-visualizer and ProLIF can both consume these SDFs.

**Post-EquiScore download (L3 Principle 14):** Download SDF files generated by EquiScore pocket extraction — these are Category A files needed for interaction analysis and user verification.

## Module 2: Interaction Analysis (interaction-visualizer — PRIMARY)

For each top-ranked candidate from Module 1, run `molclaw-interaction-visualizer` to produce deep per-structure interaction analysis, Schrödinger-style visualization, and decision-ready JSON.

**Interaction-visualizer detects:** hydrogen bonds (donor/acceptor), hydrophobic contacts, π-π stacking (face-to-face / edge-to-face), salt bridges, cation-π, halogen bonds, metal coordination, van der Waals contacts (9 types total).

**Workflow:**

1. Select top-ranked candidates from Module 1 (or Module 3 if running after consensus).
2. For each candidate, run the interaction visualizer on the docking pose:
   ```bash
   python molclaw_interaction_visualizer.py \
       --receptor prepared.pdb --ligand best_pose.sdf \
       --mode ligand --out_dir viz_out \
       --resid_offset <offset> --score <vina_score> \
       --residue_roles_json roles.json \
       --title "Target–Compound_N"
   ```
3. **Download ALL output PNGs** (L3 Principle 15): `diagram2d_*.png`, `residue_bar_*.png`, and `pymol_*_{front,side,top}.png` if PyMOL is available.
4. Parse `summary_*.json` for agent decision integration:
   - `top_residues` → verify expected key contacts match task requirements
   - `hot_partner_sites` → identify ligand atoms to preserve vs. modify (directly feeds L2-05 Step 2 diagnosis)
   - `interaction_type_counts` → quick sanity check (e.g., 0 HBonds to hinge region → flag for re-evaluation)

### Residue Numbering (L3 Principle 17 — MANDATORY when task references specific residues)

Use `--resid_offset N` (N = UniProt_number − PDB_number) so all CSV outputs include the mapped `rec_resid_mapped` column. Compute the offset from `residue_mapper.py` before invoking.

CORRECT: "Interaction visualizer detected HBond at MET769 (PDB) = Met793 (UniProt, offset +24). This confirms the expected hinge interaction."
WRONG: "Interaction visualizer did not find Met793." (False negative from numbering mismatch — check `rec_resid_mapped` column.)

### Post-Analysis Image Download (L3 Principle 15 — MANDATORY)

Download ALL interaction-visualizer output images:
- `diagram2d_*.png` — Schrödinger-style 2D interaction diagram
- `residue_bar_*.png` — residue stacked bar chart
- `pymol_*_{front,side,top}.png` — 3D multi-angle renderings (if PyMOL available)

**An interaction analysis step is NOT considered complete until all visualization images and the summary JSON have been downloaded.**

**Analysis dimensions (from `summary_*.json`):**

| Source | Key Data | Interpretation |
|--------|----------|----------------|
| `top_residues` | Residues with highest interaction count | Pharmacophore essentials — new designs MUST retain contacts with these |
| `hot_partner_sites` | Ligand atoms with high interaction count | Critical pharmacophore points to preserve during optimization |
| `interaction_type_counts` | Distribution of interaction types | A kinase inhibitor with 0 hinge HBonds is suspicious |
| `*_partner_site.csv` | Per-ligand-atom interaction hotspot | Atoms with low count are safe modification candidates |

## Module 2B: ProLIF Batch Fingerprint Analysis (SUPPLEMENT — for cross-pose comparison only)

Use ProLIF **only when** batch docking fingerprint comparison across multiple poses/molecules is needed (e.g., comparing interaction patterns of 10+ candidates simultaneously). For single-structure analysis, always use Module 2 (interaction-visualizer) above.

Call `prolif_docking` (for batch docking poses) or `prolif_pdb` (for single structures when interaction-visualizer is unavailable).

**When to use Module 2B:**
- Batch comparison of interaction fingerprints across ≥ 5 docking poses
- Need ProLIF-format fingerprint CSV for downstream pipeline compatibility
- interaction-visualizer script is not available on the compute node

**Residue numbering for ProLIF (L3 Principle 17):** ProLIF does NOT support `--resid_offset`. You must manually map ProLIF residue IDs using the mapping table from `residue_mapper.py`.

### Post-ProLIF Image Download (L3 Principle 15 — MANDATORY if Module 2B used)

Download ALL ProLIF visualization outputs:
- Interaction heatmap (PNG/SVG) — cross-pose interaction comparison
- Frequency barplot (PNG/SVG) — interaction conservation across candidates

**Cross-pose analysis dimensions (ProLIF-specific):**

| Category | Frequency threshold | Interpretation |
|----------|-------------------|----------------|
| Anchor interactions | Present in > 70% of top candidates | Pharmacophore essentials — new designs MUST retain these |
| Differentiating interactions | 30–70% | Structural basis for binding strength differences |
| Unique interactions | < 30% | Individual molecules with special binding modes |

## Module 2C: FoldX Interface Energy (Optional — Protein-Protein/Peptide-Protein Only)

For protein-protein or peptide-protein complexes (NOT small-molecule ligands), FoldX AnalyseComplex provides interface interaction energy as an additional physics-based validation dimension.

**When to use Module 2C:**
- The docked complex is a protein-protein or peptide-protein system
- A quantitative energy ranking is desired beyond geometric interaction analysis (Module 2)
- MMPBSA (Skill 6) would be too expensive at this screening stage

**When NOT to use:**
- Small-molecule ligand complexes (FoldX AnalyseComplex is designed for protein-protein interfaces; for small molecules use Boltz-2 or MMPBSA)
- The complex has not been structurally repaired for FoldX

**Procedure:**

1. Run FoldX RepairPDB on each candidate complex.
2. Run FoldX AnalyseComplex:

```python
response = await client.session.call_tool("foldx_tool", arguments={
    "mode": "analysecomplex",
    "pdb_path": foldx_repaired_complex_path,
    "chains": "A,B"   # MUST match actual chain IDs in PDB
})
result = client.parse_result(response)
interaction_energy = result["metrics"]["interaction_energy"]  # kcal/mol
```

3. Rank candidates by interaction_energy (more negative = stronger interface).
4. Include FoldX interaction energy as an additional column in Module 3 consensus ranking.

**Scoring interpretation:** FoldX interaction energy is an empirical estimate with ±0.5–1.0 kcal/mol uncertainty. Use for ranking, not absolute affinity. Do not convert to Kd.

## Module 3: Consensus Ranking

**Step 1 — Data assembly.** Merge all scoring dimensions into a unified matrix.

**⚠ COUNT GATE (L3 Principle 11):** Before assembling the consensus table, verify that the number of molecules in each scoring dimension matches. If EquiScore scored 18 molecules but Vina scored 20, identify and explain the 2 missing molecules (EquiScore failure? Format issue?).

**Step 2 — Normalization.** Each scoring dimension has different scales and directions. Normalize all to 0–1 (higher = better):
- Vina (more negative = better): `norm = (max - x) / (max - min)`
- EquiScore (higher = better): `norm = (x - min) / (max - min)`
- ProLIF interaction count: normalize by maximum observed count
- Boltz-2 probability: already 0–1

**Step 3 — Weighted consensus score.**

| Dimension | Default weight | Rationale |
|-----------|---------------|-----------|
| Vina | 0.30 | Established physics-based scoring |
| EquiScore | 0.30 | ML-based, complementary to Vina |
| ProLIF | 0.25 | Interaction quality, not just total score |
| Boltz-2 | 0.15 | Independent affinity estimate |

If a dimension is missing, redistribute its weight proportionally. Document the rationale.

**Step 4 — Consistency analysis.** For each molecule, compare its rank across methods:
- **Level A (high consistency):** Top 25% by ALL methods → most trustworthy
- **Level B (moderate):** Top 25% by most methods → reliable
- **Level C (divergent):** Methods strongly disagree → inspect docking pose; re-dock if needed

## Module 4: Preliminary SAR Analysis (optional)

When candidates share a common scaffold, combine interaction-visualizer CSV data (or ProLIF fingerprints if Module 2B was used) with property data for structure-activity insights.

**Computation-first rule (L3 Principle 13):** SAR insights must be grounded in tool-computed data (interaction-visualizer CSV/JSON, ProLIF fingerprints, computed property differences, docking score variations). Do NOT generate SAR conclusions from LLM knowledge about drug-target interactions. CORRECT: "Replacing the methyl at C3 with fluoro improved docking by 0.8 kcal/mol; interaction visualizer shows a new halogen bond with Thr790 (UniProt, rec_resid_mapped) = Thr122 (Boltz-2, rec_resid_pdb)." WRONG: "Fluorine substitution typically improves binding to kinase hinge regions" (this is LLM knowledge, not a tool-computed observation about THIS specific case).

**Similarity clustering:** Use `calculate_morgan_fingerprint_similarity`. Group molecules with Tanimoto > 0.80 into the same scaffold series.

**Fingerprint differential analysis within a series:** Compare interaction profiles (from interaction-visualizer `residue_summary.csv` or ProLIF fingerprints) and docking scores among series members. Identify "what structural change led to what binding change."

**Property-activity correlation:** Check if LogP, MW, or other properties correlate with docking scores.

## Common Failures & Recovery

| Failure | Likely cause | Recovery |
|---------|-------------|----------|
| EquiScore fails on some poses | Ligand format issue; atoms outside pocket box | Re-extract pocket with larger radius |
| Interaction-visualizer detects zero interactions | Ligand not in pocket; wrong `--ligand_resname`; bad docking pose | Verify docking pose quality; check `--ligand_resname` matches HETATM resname |
| Interaction-visualizer residue IDs don't match task | `--resid_offset` not set or wrong | Compute offset from `residue_mapper.py` and re-run |
| ProLIF detects zero interactions (Module 2B) | Ligand not in pocket; or format conversion lost coordinates | Verify docking pose; re-convert PDBQT → SDF |
| ProLIF residue IDs don't match task (Module 2B) | Numbering scheme mismatch | **Execute residue mapping (L3 Principle 17)** |
| Consensus ranking contradicts all individual methods | Normalization issue with outliers | Remove outliers and re-normalize |
